🚨 ACİL RLS DEVRE DIŞI BIRAKMA

⚠️ ÖNEMLİ: Bu geçici bir çözümdür!

RLS'yi devre dışı bırakarak uygulamanın hemen çalışmasını sağlayacağız. Bu güvenlik açısından ideal değildir ama uygulamanızı hemen kullanabilir hale getirecektir.

🔧 Hızlı Çözüm Adımları:

1. Supabase Dashboard'a gidin: https://supabase.com/dashboard/project/jxbfdgyusoehqybxdnii
2. Sol menüden "SQL Editor" seçin
3. Aşağıdaki "KOPYALA" butonuna tıklayın
4. SQL Editor'e yapıştırın ve "Run" butonuna tıklayın
5. Uygulamayı yenileyin ve test edin
-- ACİL RLS DEVRE DIŞI BIRAKMA - Uygulamayı hemen çalışır hale getir -- 1. Tüm tablolarda RLS'yi devre dışı bırak ALTER TABLE public.birds DISABLE ROW LEVEL SECURITY; ALTER TABLE public.chicks DISABLE ROW LEVEL SECURITY; ALTER TABLE public.eggs DISABLE ROW LEVEL SECURITY; ALTER TABLE public.incubations DISABLE ROW LEVEL SECURITY; ALTER TABLE public.user_notification_settings DISABLE ROW LEVEL SECURITY; ALTER TABLE public.user_notification_tokens DISABLE ROW LEVEL SECURITY; ALTER TABLE public.notification_interactions DISABLE ROW LEVEL SECURITY; ALTER TABLE public.profiles DISABLE ROW LEVEL SECURITY; ALTER TABLE public.photos DISABLE ROW LEVEL SECURITY; ALTER TABLE public.feedback DISABLE ROW LEVEL SECURITY; ALTER TABLE public.calendar DISABLE ROW LEVEL SECURITY; ALTER TABLE public.backup_jobs DISABLE ROW LEVEL SECURITY; ALTER TABLE public.backup_settings DISABLE ROW LEVEL SECURITY; ALTER TABLE public.backup_history DISABLE ROW LEVEL SECURITY; ALTER TABLE public.clutches DISABLE ROW LEVEL SECURITY; -- 2. Bildirim ayarlarını mevcut kullanıcılar için oluştur INSERT INTO public.user_notification_settings (user_id) SELECT id FROM auth.users ON CONFLICT (user_id) DO NOTHING; -- 3. Durumu kontrol et SELECT schemaname, tablename, rowsecurity FROM pg_tables WHERE schemaname = 'public' AND tablename IN ('birds', 'chicks', 'eggs', 'incubations', 'user_notification_settings') ORDER BY tablename;

✅ Bu SQL kodu şunları yapacak:

  • 🔓 Tüm tablolarda RLS'yi devre dışı bırakır
  • ⚡ Uygulamanızı hemen çalışır hale getirir
  • 📧 Mevcut kullanıcılar için bildirim ayarları oluşturur
  • 🔍 RLS durumunu kontrol eder

⚠️ Güvenlik Notu:

Bu geçici bir çözümdür. RLS devre dışı olduğunda:

  • Kullanıcılar birbirlerinin verilerini görebilir
  • Güvenlik seviyesi düşer
  • Daha sonra RLS'yi tekrar aktif etmeniz önerilir

🔍 Test Etmek İçin:

SQL çalıştırdıktan sonra uygulamada:

  1. Yeni bir kuş eklemeyi deneyin (SUNNY, BELLA, MANGO vb.)
  2. Bildirim ayarlarını kaydetmeyi deneyin
  3. Herhangi bir hata mesajı olup olmadığını kontrol edin
  4. Kuşların listelendiğini kontrol edin

🔄 Daha Sonra RLS'yi Tekrar Aktif Etmek İçin:

Uygulama çalışır hale geldikten sonra, RLS'yi güvenli bir şekilde tekrar aktif etmek için:

  1. RLS politikalarını düzgün bir şekilde yapılandırın
  2. Test edin
  3. RLS'yi tekrar aktif edin
